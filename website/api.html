API Reference

REST endpoints for workflows, simulator control, speed benchmark, and individual agents.

Health & settings

MethodEndpointDescription
GET/api/healthServer status, LLM provider, model
GET/api/settingsActive provider configuration
GET/api/models/cerebrasGemma 4 model info

Workflow

MethodEndpointDescription
POST/api/runsCreate run — body: instruction, scene_id
GET/api/runsList all runs
GET/api/runs/{id}Run detail + agent outputs
POST/api/runs/{id}/uploadUpload image and/or MP4 (multipart)
POST/api/runs/{id}/executeStart agent pipeline
POST/api/runs/{id}/confirmHuman-in-the-loop safety override
POST/api/runs/{id}/rerunRe-run workflow
GET/api/runs/{id}/eventsLive event timeline
GET/api/runs/{id}/reportReport — ?format=markdown

Speed benchmark

POST /api/benchmark/speed?iterations=3

Runs repeated Vision Lab calls. Returns avg_ms, min_ms, max_ms, headline.

{
  "provider": "cerebras-gemma4",
  "model": "gemma-4-31b",
  "avg_ms": 847.5,
  "headline": "Gemma 4 on Cerebras: 848ms avg vision call"
}

Example: full workflow

# 1. Create run
curl -X POST http://127.0.0.1:7860/api/runs \
  -H "Content-Type: application/json" \
  -d '{"instruction":"Move red vial to safe tray","scene_id":"safe_sorting_scene"}'

# 2. Upload image
curl -X POST http://127.0.0.1:7860/api/runs/{id}/upload \
  -F "image=@bench.png"

# 3. Execute
curl -X POST http://127.0.0.1:7860/api/runs/{id}/execute

# 4. Poll status
curl http://127.0.0.1:7860/api/runs/{id}
