Architecture

How VialPilot's nine agents, multimodal vision, replan loop, and 3D simulator work together.

System overview

FastAPI backend orchestrates nine specialist agents. Gemma 4 on Cerebras handles all LLM calls. Commands flow to a Python robot state machine, synced to a Three.js WebGL arm in the browser.

Replan loop

Each subtask runs plan → act → reflect. If Reflector returns retry_needed: true, Motion Planner re-runs with a hint — maximum one replan per subtask.

Safety allow → Motion Planner → Actor Command
    → Simulator moves arm → Post-action PNG
    → Reflector (Gemma 4 + image)
    → if retry_needed: replan_started → Planner(hint) → Actor → Reflector
    → replan_completed → continue

LLM provider chain

run_json() per agent call
  1. Cerebras Gemma 4 31B (multimodal, JSON mode)
  2. Gemini 2.5 Flash (fallback)
  3. Mock offline (tests, no API key)

Every call records latency_ms and mode (real / mock) for the Speed in Action panel.

3D simulator

  • Backend: SoftwareRobotBackend — deterministic state machine, no PyBullet required on Windows
  • API: GET /simulator/scene, POST /simulator/step
  • Frontend: lab3d.js — Three.js IK, joint interpolation, arc sweep pick/place
  • Sync: Browser polls scene JSON; arm animates via animateSweepPick/Place

Data model

SQLite via SQLAlchemy. Key tables: runs (instruction, agent_outputs, commands, latency_metrics, bench_state) and run_events (timeline: replan_started, agent_completed, etc.).
